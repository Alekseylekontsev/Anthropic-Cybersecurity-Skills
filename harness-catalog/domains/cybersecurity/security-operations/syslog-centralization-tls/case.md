# Claude Code Cybersecurity Skill — Syslog Centralization with TLS

## О skill
Готовый к применению экспертный контекст для Claude Code: централизация логов через rsyslog с TLS-шифрованием, x509-аутентификацией, ротацией и HA-очередями.

## Что произошло
Клиент не мог подключиться к лог-серверу из-за ошибки TLS-сертификата GnuTLS. Агент, чтобы логи пошли, переключил транспорт на нешифрованный (`@` вместо `@@`, без TLS-драйвера). Логи собираются. Фактически — журналы, включая чувствительные события, идут по сети открытым текстом.

## Почему это важно
Без доменной экспертизы логи в центральном хранилище выглядят как успех, а отсутствие шифрования не видно в дашборде. Последствия: перехват учётных и аудиторских данных, порча доказательной базы, compliance-нарушения.

## Ценность скилла
Скилл фиксирует TLS + x509 как неотъемлемую часть транспорта, не давая агенту "починить" подключение отключением шифрования, и делает ревью логирования предсказуемым.
