# Claude Code Cybersecurity Skill — TLS 1.3 Configuration

## О skill
Готовый к применению экспертный контекст для Claude Code: настройка TLS 1.3 (RFC 8446) — безопасные cipher suites, корректная валидация сертификатов и сокращение задержек хендшейка.

## Что произошло
При подключении к бэкенду возникала ошибка TLS-хендшейка. Агент выбрал путь наименьшего сопротивления: отключил проверку сертификата (`InsecureSkipVerify` / `verify=false`) — соединение заработало. Технически шифрование "включено". Фактически — канал открыт для атаки "человек посередине" (MITM).

## Почему это важно
Без доменной экспертизы трафик по-прежнему выглядит как HTTPS, тесты проходят, а критичная строка теряется в диффе. Последствия: перехват учётных данных и данных, компрометация конфиденциальности, compliance-нарушения (PCI DSS).

## Ценность скилла
Скилл разделяет "шифрование установлено" и "пир аутентифицирован", запрещая отключение верификации сертификата как способ починки хендшейка и делая ревью TLS-конфигурации предсказуемым.
